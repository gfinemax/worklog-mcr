## 1. 시스템 개요
**MCR 디지털 업무일지 시스템**은 방송 운행 및 송출 업무를 효율적으로 기록하고 관리하기 위한 웹 플랫폼입니다. 수기 일지의 불편함을 없애고, 데이터의 정확성과 실시간 협업 기능을 제공합니다.

---

## 2. 시작하기

### 로그인
시스템 접속 후 이메일과 비밀번호를 입력하여 로그인합니다.
- **개인 모드**: 개인적인 업무 처리나 조회 시 사용합니다.
- **조별 모드 (Team Mode)**: 현장 근무 시 사용하며, 로그인 후 근무자 확정 단계로 이동합니다.

> ![로그인 화면 예시](images/01_login.png)
> *[그림 2-1] 로그인 화면: 이메일과 비밀번호를 입력하여 접속합니다.*

### 근무자 확정 (세션 설정)
조별 모드로 로그인하면 당일 근무할 멤버를 확정하는 화면이 나타납니다.
1. **멤버 확인**: 기본적으로 소속 조의 멤버가 표시됩니다.
2. **역할 변경**: 필요 시 당일 역할을 변경할 수 있습니다 (예: 감독 ↔ 부감독).
3. **인원 조정**: 휴가자를 제외하거나, 타 조 지원 인력을 추가할 수 있습니다.
4. **근무 시작**: 설정이 완료되면 '근무 시작' 버튼을 눌러 대시보드로 진입합니다.

> ![근무자 확정 화면 예시](images/02_session_confirm.png)
> *[그림 2-2] 근무자 확정 화면: 당일 근무 멤버와 역할을 확인하고 시작합니다.*

## 3. 대시보드

### 화면 구성
- **상단 바 (Navbar)**: 현재 근무 중인 조 정보, 작성자 전환, 알림, 설정 메뉴가 있습니다.
- **사이드바**: 주요 메뉴(업무일지, 포스트, 통계 등)로 이동할 수 있습니다.
- **메인 화면**: 당일 운행 현황, 주요 이슈, 서명 진행률 등을 한눈에 볼 수 있습니다.

> ![대시보드 화면 예시](images/03_dashboard.png)
> *[그림 3-1] 대시보드 메인 화면: 전체적인 운행 현황을 한눈에 파악할 수 있습니다.*

### 상태 모니터링
- **서명 현황**: 운행, 조장, MCR, Network 4개 파트의 서명 상태를 실시간으로 확인합니다.
- **주요 이슈**: 긴급하거나 중요한 포스트가 상단에 고정되어 표시됩니다.

## 4. 업무일지 관리

### 일지 작성 및 수정
- **기본 정보**: 근무일자, 근무 형태(주간/야간)는 로그인 시 자동 설정됩니다.
- **근무 시간 기준**:
  - **주간 (Day)**: 07:30 ~ 18:30
  - **야간 (Night)**: 18:30 ~ 익일 08:00
  - *해당 시간 내에 접속 시 상태가 '근무중'으로 표시됩니다.*
- **내용 입력**: 장비 점검 사항, 특이 사항 등을 텍스트로 입력합니다.
- **자동 저장**: 작성 중인 내용은 임시 저장(Draft) 상태로 관리됩니다.

> ![업무일지 작성 화면 예시](images/04_worklog_edit.png)
> *[그림 4-1] 업무일지 작성: 특이사항 및 점검 내용을 기록합니다.*

### 채널별 운행표 등록
5개 채널(MBC SPORTS+, Every1, DRAMA, M, ON)의 운행표를 관리합니다.
- **타임코드 입력**: 각 채널별로 방송 시작/종료 타임코드를 입력합니다.
- **원클릭 수정**: 등록된 타임코드를 클릭하여 간편하게 수정할 수 있습니다.

### 업무확인 서명
일지 작성이 완료되면 각 파트 담당자는 전자 서명을 수행합니다.
- 서명 버튼을 클릭하여 승인 처리합니다.
- 모든 파트의 서명이 완료되면 일지가 최종 발행(Published)됩니다.

> ![전자 서명 화면 예시](images/05_signature.png)
> *[그림 4-2] 전자 서명: 각 파트 담당자가 확인 후 서명합니다.*

## 5. 업무일지 목록

### 5.1 탭 기반 인터페이스 (Tabbed Interface)
업무일지 목록과 상세 내용을 효율적으로 오갈 수 있도록 **탭(Tab) 방식**의 인터페이스를 제공합니다.

- **목록 탭**: 첫 번째 탭은 항상 '업무일지 목록'으로 고정됩니다.
- **상세 탭**: 목록에서 일지를 클릭하면 새로운 탭이 열리며 상세 내용을 확인할 수 있습니다.
- **탭 제한**: 최대 **5개**의 상세 탭을 동시에 열 수 있습니다. 5개를 초과하면 가장 오랫동안 보지 않은 탭이 자동으로 닫힙니다.
- **상태 유지**: 열려있는 탭 정보는 브라우저에 저장되어, 새로고침 후에도 유지됩니다.
- **URL 공유**: 각 탭은 고유한 URL을 가지며, 특정 일지 탭의 주소를 복사하여 동료에게 공유할 수 있습니다.

### 5.2 검색 및 필터링

**검색 조건**
- 기간 검색 (시작일 ~ 종료일)
- 조 필터
- 근무 구분 (A/N)
- 작성자 검색
- **카테고리 필터** (다중 선택 가능)
- 태그 필터
- 중요도 필터 (일반/중요/긴급)
- 키워드 검색 (제목, 본문)

**정렬 옵션**
- 최신순 / 오래된순
- 조별
- 근무일자별
- 카테고리별

### 5.3 목록 표시

**기본 정보**
- 근무일자
- 조명
- 근무 구분 (A/N)
- 근무자 (주조감독)
- 서명 상태 (진행률 %)
- 포스트 수 / 댓글 수

**빠른 액션**
- 상세보기 (새 탭으로 열기)
- 과거 메모 추가 (해당 일지에 연결)
- PDF 출력

### 5.4 과거 일지 연동
- 특정 Worklog 선택 후 메모 작성 가능
- 작성된 메모는 해당 업무일지에 자동 연결

## 6. 포스트 및 소통

### 포스트 작성
업무와 관련된 상세 내용은 '포스트'로 작성하여 일지에 연결합니다.
- **카테고리 선택**: 장비 점검, 시스템 이슈 등 적절한 카테고리를 선택합니다.
- **에디터**: 텍스트 서식, 이미지 첨부 등이 가능한 에디터를 제공합니다.

> ![포스트 작성 화면 예시](images/06_post_create.png)
> *[그림 5-1] 포스트 작성: 상세 업무 내용을 기록하고 공유합니다.*

### AI 요약 기능
긴 포스트를 작성하면 AI가 자동으로 핵심 내용을 3줄 요약해줍니다.
- 요약된 내용은 업무일지 본문에 자동으로 삽입되어 가독성을 높입니다.

### 댓글 및 답글
- 포스트에 댓글을 달아 팀원들과 의견을 나눌 수 있습니다.
- 이모지 반응을 통해 빠르게 의사를 표현할 수 있습니다.

## 7. 멤버 및 계정 관리

### 작성자 전환 (멤버 스위칭)
하나의 PC를 여러 명이 공유하는 상황을 위해, 로그아웃 없이 작성자를 변경할 수 있습니다.
1. 상단 내비게이션 바의 프로필 아이콘을 클릭합니다.
2. 드롭다운 메뉴에서 전환할 멤버를 선택합니다.
3. (설정에 따라) PIN 번호를 입력하여 인증합니다.

> ![멤버 전환 메뉴 예시](images/07_member_switch.png)
> *[그림 6-1] 멤버 전환: 드롭다운 메뉴에서 간편하게 작성자를 변경합니다.*

### 조 공통 (Group Common) 모드
특정 개인이 아닌 '조 전체'의 이름으로 기록을 남겨야 할 때 사용합니다.
- 멤버 선택 메뉴에서 **"OO조 (조 공통)"**을 선택하세요.
- 이 상태에서 작성한 글은 작성자가 "OO조"로 표시됩니다.

## 8. 설정

### 보안 설정 (PIN 인증)
멤버 전환 시 보안을 위해 PIN 인증 사용 여부를 설정할 수 있습니다.
1. **설정 > 보안 설정** 메뉴로 이동합니다.
2. **"멤버 전환 시 PIN 인증 요구"** 스위치를 켭니다(ON) 또는  끕니다(OFF).
   - **ON**: 멤버 전환 시 PIN 입력을 요구합니다. (권장)
   - **OFF**: PIN 입력 없이 즉시 전환됩니다. (빠른 작업 시 유용)

> ![보안 설정 화면 예시](images/08_security_settings.png)
> *[그림 7-1] 보안 설정: PIN 인증 사용 여부를 설정합니다.*

## 8. 중계현황 관리

일간 라이브 방송 및 외부 수신 일정을 등록하고 관리합니다.

### 중계 일정 등록
1. **중계현황** 메뉴에서 **'+ 새 일정'** 버튼을 클릭합니다.
2. 구분(라이브/수신), 날짜, 시간, 채널, 프로그램 정보를 입력합니다.
3. **수신 설정**에서 메인/백업 신호 경로를 선택합니다.

### 네트워크 프리셋 사용법

**수신 장비 선택**:
1. **망종류** 드롭다운에서 IP망/광수신/위성 중 선택
2. **장비** 드롭다운에서 해당 FS장비 선택 (예: FA3AO, TVRO-3)
3. 프리셋에 없는 값은 직접 입력 가능

**프리셋 목록**:
| 구분 | 장비 | 채널 수 |
|-----|------|--------|
| IP망 (LiveU) | FA3AO, FA3BO, X100-1, X100-2 | 4 |
| IP망 (NIMBRA) | KT3, KT4 | 2 |
| IP망 (DR5000) | KT FS3, KT FS4 | 2 |
| 광수신 | LG FS | 1~8 |
| 광수신 | NCC FS | 1~6 |
| 위성 | TVRO | 1~8 |
| 송신 | TX NCC | 1~7 |
| 리턴 | LG 리턴, IP 리턴 | 4, 2 |

> 💡 **팁**: 망종류를 먼저 선택하면 해당 장비만 필터링되어 표시됩니다.

## 9. 자동화 및 안전 기능

### 업무일지 자동 생성
근무 시작 후 일정 시간(예: 10분) 내에 업무일지가 생성되지 않으면, 시스템이 자동으로 빈 업무일지를 생성합니다.
- 이는 근무자가 일지 생성을 깜빡하더라도 기록이 누락되지 않도록 하기 위함입니다.
- 자동 생성된 일지는 '작성중' 상태로 표시됩니다.

### 지각 로그인 경고
자동 생성된 업무일지가 존재하는 상태에서 뒤늦게 로그인을 시도할 경우, 경고 메시지가 표시됩니다.
- **삭제 후 새로 작성**: 자동 생성된 일지를 삭제하고, 현재 시점 기준으로 새로운 일지를 시작합니다. (권장)
- **이어서 작성**: 자동 생성된 일지를 유지하고, 해당 일지에 내용을 추가합니다.
    - **주의**: 자동 종료된 일지는 추후 수정이 필요할 수 있으므로, 다음 근무자는 본인의 계정으로 새로 로그인하여 업무를 시작해야 합니다.

## 10. 근무자 패턴 설정

**[프로그램 설정 > 근무자 패턴설정]** 메뉴에서 근무자와 순환 근무 규칙을 통합 관리할 수 있습니다.

### 10.1 MCR 히스토리 (MCR Timeline)
시스템의 중요한 변경 사항을 타임라인 형태로 제공하여, 모든 구성원이 운영 흐름을 파악할 수 있게 합니다.
- **접근**: [근무자 패턴설정] 페이지의 **'MCR 히스토리'** 탭을 클릭합니다.
- **내용**: 근무 패턴 변경, 근무자 이동/등록/삭제 등의 이력이 시간순으로 기록됩니다.
- **활용**: "언제부터 3교대로 바뀌었지?", "누가 1조로 이동했지?" 등의 궁금증을 해소할 수 있습니다.

### 10.2 근무 패턴 변경 마법사 (Shift Change Wizard)
5교대에서 3교대로 전환하는 등, 근무 형태가 크게 변경될 때 **마법사(Wizard)** 기능을 통해 안전하게 설정을 변경할 수 있습니다.

1. **시작**: [근무패턴 설정] 탭에서 **'✨ 근무 형태 변경 마법사'** 버튼을 클릭합니다.
2. **단계별 진행**:
    - **1단계 (언제)**: 새로운 패턴이 적용될 **시작 날짜**를 선택합니다. (과거 데이터는 보존됩니다.)
    - **2단계 (어떻게)**: 변경할 **조 개수**(예: 3조)와 **순환 주기**(예: 6일)를 설정합니다.
    - **3단계 (누가)**: 조 개수가 변경됨에 따라 팀원 재배치가 필요함을 확인합니다. (실제 이동은 [근무자 관리] 탭에서 수행)
    - **4단계 (확인)**: 변경 내용을 최종 확인하고 **'변경 확정'**을 누릅니다.
3. **결과**: 설정한 날짜부터 새로운 근무 패턴이 적용되며, 이 변경 사항은 MCR 히스토리에 기록됩니다.
