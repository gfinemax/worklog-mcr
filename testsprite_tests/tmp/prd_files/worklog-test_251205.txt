1. 인증 및 세션 관리 (Authentication & Session)
가장 중요하고 최근 이슈가 많았던 부분입니다. 로그인 상태 유지와 리다이렉션 로직을 중점적으로 확인해야 합니다.

주요 파일:
app/login/page.tsx
: 로그인 폼 UI 및 핸들링.
components/auth-provider.tsx
: 전역 인증 상태 관리 (Session Context).
store/auth.ts
: Zustand 인증 스토어 (SessionStorage 사용 부분 주의).
lib/supabase.ts
: 클라이언트 사이드 Supabase 클라이언트 생성 로직.
디버깅 체크포인트:
세션 유지: 새로고침 시 로그인이 풀리지 않고 auth-provider가 세션을 잘 복구하는지.
리다이렉트 루프: 로그인 페이지와 메인 페이지 간의 무한 리다이렉트 발생 여부 (특히 Google 로그인 시).
PIN 인증: app/settings에서 설정한 PIN 번호가 로그인 시 정상적으로 요구되고 검증되는지.
SSR 불일치: 서버 컴포넌트와 클라이언트 컴포넌트 간의 인증 상태 불일치로 인한 Hydration Error 확인.
2. 업무일지 및 근무표 (Worklog & Shift Management)
비즈니스 로직이 가장 복잡한 핵심 기능입니다. 데이터 동기화와 복잡한 UI 상호작용을 확인해야 합니다.

주요 파일:
app/worklog/page.tsx 및 하위 페이지들.
store/worklog.ts: 업무일지 데이터 관리 (가장 복잡함).
store/worklog-tab-store.ts: 탭 네비게이션 상태 관리.
components/worker-registration-dialog.tsx: 근무자 등록 팝업 (코드량이 많아 버그 가능성 높음).
components/worklog/: 업무일지 관련 컴포넌트들.
디버깅 체크포인트:
탭 네비게이션: 업무일지 목록에서 클릭 시 새 탭이 열리고, URL이 정상적으로 업데이트되며, 탭 간 전환 시 데이터가 유지되는지.
근무자 매칭: 로그인한 사용자의 역할(감독/조감독)이 업무일지에 올바르게 자동 기입되는지 (최근 버그 있었음).
Shift Wizard: 근무표 생성 마법사(Step 1~3)에서 데이터가 누락되거나 "Unassigned" 목록이 비어있는 현상 확인.
데이터 저장: 폼 입력 중 자동 저장이나 수동 저장 시 데이터 유실이 없는지.
3. 게시판 및 소통 (Posts & Community)
최근 AI 요약 및 에디터 기능이 추가된 영역입니다.

주요 파일:
app/posts/[id]/page.tsx: 게시글 상세 및 댓글.
components/post-editor.tsx: 게시글 작성 에디터 (Quill Editor).
store/posts.ts: 게시글 상태 관리.
디버깅 체크포인트:
에디터 이미지: react-quill-new 사용 시 이미지 리사이징 및 업로드가 정상 작동하는지.
AI 요약: 게시글 AI 요약 생성 시 에러(404, 429 등) 없이 결과가 표시되는지.
댓글 이모지: 커스텀 이모지 피커가 정상적으로 뜨고, 선택한 이모지가 댓글에 잘 반영되는지.
낙관적 업데이트: 댓글 작성/삭제 시 화면에 즉시 반영되고 실패 시 롤백되는지.
4. 중계 현황 (Broadcasts)
새로 기획/추가 중인 기능으로, 데이터 표시와 레이아웃을 점검해야 합니다.

주요 파일:
app/broadcasts/page.tsx: 중계 리스트.
app/broadcasts/[id]/monitor/page.tsx: 상세 모니터링 뷰.
디버깅 체크포인트:
레이아웃 반응형: 모니터링 뷰가 전체 화면 모드일 때 레이아웃이 깨지지 않는지.
데이터 필터링: 중계/수신 일정 분류가 디자인대로 색상 코딩되어 표시되는지.
5. UI/UX 및 공통 사항 (Global UI/UX)
전반적인 사용자 경험과 성능을 점검합니다.

주요 파일:
app/layout.tsx: 전역 레이아웃.
app/globals.css: 전역 스타일 (Tailwind).
components/ui/: Shadcn UI 컴포넌트들.
디버깅 체크포인트:
다크 모드: next-themes 전환 시 깜빡임이나 스타일 깨짐이 없는지.
모바일 뷰: 사이드바(app/(dashboard)/layout.tsx)가 모바일에서 정상적으로 열리고 닫히는지.
에러 핸들링: API 호출 실패 시 사용자에게 적절한 토스트 메시지(sonner)가 뜨는지.
